# Main Analysis Repository

This repository contains the primary analysis code and data used in the study, including the main notebook for data processing and figure generation, the raw dataset used in the analyses, a cached table of logistic regression results, and a Fiji/ImageJ macro for image-based feature extraction.

## Overview

The repository serves as the main computational workspace for the study. It brings together the core data table, the analysis notebook used for data manipulation and figure generation, and supporting files required for feature extraction and statistical modeling.

This repository is best interpreted as a study-focused analysis and reproducibility resource rather than a general-purpose software package.

## Purpose

The repository supports the main quantitative analysis workflow of the project. It includes code for preparing and analyzing the dataset, generating figures, and evaluating logistic regression models across multiple combinations of features.

It also includes the macro used to extract image-derived features from masks before those measurements are incorporated into the downstream analyses.

## Repository contents

### `main_data_analysis.ipynb`

Primary analysis notebook containing the data manipulations, statistical analyses, and figure generation used in the study.

### `Main_Data.csv`

Raw data table used as the starting point for the main analyses.

### `logistic_regression_results_all_days_featuresets.csv`

Precomputed results table generated by the notebook from multiple logistic regression runs across a large set of feature combinations. This file is included to improve reproducibility and save time, since users do not need to rerun the full combinatorial modeling workflow. Markdown in notebook identifies the cells to skip.

### `Feature_extraction.ijm`

Fiji/ImageJ macro used to extract quantitative features from image masks for inclusion in the analysis pipeline. Works with the masks generated by the segmentation code.

## How the workflow works

The analysis begins with the raw data stored in `Main_Data.csv`, that has the image derived features captured with `Feature_extraction.ijm` and metabolite exchange rates obtained as described in the methods session of the original paper. These data are then processed and analyzed in `main_data_analysis.ipynb`, where the main transformations, model evaluations, and figure generation steps are performed.

The logistic regression results file is provided as a cached output from an extensive model-comparison step. Including this file makes it possible to inspect or reuse those results directly without repeating the full set of model runs.

## Intended use

This repository accompanies the manuscript submission and is intended primarily as a reproducibility and analysis resource. It documents the data-processing and figure-generation workflow used in the study and preserves intermediate outputs that would otherwise be computationally repetitive to recreate.

## Notes for reuse

Users reusing this repository should note that some outputs are already precomputed for convenience. In particular, `logistic_regression_results_all_days_featuresets.csv` is included to avoid unnecessary reruns of the full logistic regression combination analysis.

The ImageJ macro is intended for feature extraction from image masks and is part of the upstream workflow that feeds into the notebook-based analyses.
